          Điểm khác biệt
          
                        
          API này cung cấp dữ liệu được tối ưu cho phần header, bao gồm cả thông tin điều hướng. So với API company-info, API này:
          
                        
            
                            
          • Bao gồm thông tin điều hướng (navigation) sử dụng cho thanh navbar
            • 
                            
          • Lấy dữ liệu điều hướng từ global header để quản trị viên có thể dễ dàng cập nhật
            • 
                            
          • Có đầy đủ các mạng xã hội và thông tin liên hệ cần thiết
            • 
                            
          • Có định dạng tối ưu hơn cho việc sử dụng trong frontend
            • 
                        
          
                        
                        
          Sử dụng Navigation Links
          
                        
          Khi nhấn nút "Lấy dữ liệu từ API", bạn sẽ thấy phần "Xem trước Điều hướng" hiển thị danh sách các liên kết từ API theo 2 loại:
          
                        
            
                            
          • Menu Chính: Các liên kết chính hiển thị trong thanh điều hướng chính (được chuyển đổi từ navItems của global header)
            • 
                            
          • Menu Phụ: Các liên kết bổ sung có thể hiển thị trong menu dropdown hoặc phần "Xem thêm"
            • 
                        
          
                        
          Mỗi liên kết bao gồm title (văn bản hiển thị) và routeKey (khóa định danh cho route).
          
                        
                        
          Cách thức cập nhật liên kết:
          
                        
            
                            
          1. Đăng nhập vào bảng quản trị tại /admin
            2. 
                            
          2. Điều hướng tới Globals → Header
            3. 
                            
          3. Cập nhật mục Navigation Items
            4. 
                            
          4. Lưu thay đổi, các liên kết mới sẽ tự động xuất hiện trên website
            5.

          Ví dụ với Vanilla JavaScript:
          
                        
          // Lấy dữ liệu header
async function loadHeaderInfo() {
  try {
    const response = await fetch('/api/header-info');
    const data = await response.json();
    
    // Render logo
    if (data.logo && data.logo.url) {
      document.getElementById('companyLogo').src = data.logo.url;
      document.getElementById('companyLogo').alt = data.logo.alt || data.companyName;
    }
    
    // Render navigation
    const navContainer = document.getElementById('mainNav');
    data.navigation.mainLinks.forEach(link => {
      const li = document.createElement('li');
      const a = document.createElement('a');
      a.href = `/${link.routeKey.toLowerCase()}`;
      a.textContent = link.title;
      li.appendChild(a);
      navContainer.appendChild(li);
    });
  } catch (error) {
    console.error('Lỗi khi tải header:', error);
  }
}

          Xử lý lỗi CORS
          
                    
          
                        
          Vấn đề CORS phổ biến
          
                        
          Khi gọi API từ một nguồn khác với nguồn của API, có thể gặp lỗi CORS (Cross-Origin Resource Sharing).
          
                        
                        
          Lỗi khi mở file trực tiếp (file://)
          
                        
          Lỗi phổ biến: Access to fetch at 'file:///E:/api/globals/company-info' from origin 'null' has been blocked by CORS policy
          
                        
          Nguyên nhân: Trình duyệt không cho phép các yêu cầu fetch từ một file HTML được mở trực tiếp (protocol file://).
          
                        
          Giải pháp:
          
                        
            
                            
          1. Đặt file test vào thư mục backend/public và truy cập qua http://localhost:3000/api-test.html
            2. 
                            
          2. Sử dụng Live Server extension trong VS Code để chạy trên máy chủ đơn giản
            3. 
                            
          3. Sao chép file vào thư mục gốc của backend và truy cập qua http://localhost:3000
            4. 
                        
          
                        
                        
          Giải pháp cho các lỗi CORS khác
          
                        
            
                            
          1. Đảm bảo máy chủ API cấu hình CORS cho phép nguồn gốc của bạn
            2. 
                            
          2. Gọi API thông qua đường dẫn tương đối (ví dụ: /api/...) khi frontend và backend cùng domain
            3. 
                            
          3. Sử dụng proxy trong quá trình phát triển
            4. 
                        
          
                        
                        
          Cấu hình CORS trong Payload CMS
          
                        
          Xem file payload.config.ts để biết cấu hình CORS hiện tại:
          
                        
          cors: {
  origins: [
    'http://localhost:3000',  // Backend URL
    'http://localhost:5173',  // Frontend URL
    // Các domain khác
  ],
  headers: ['authorization', 'content-type', ...]
}
